Color Detection

About

As seen in the previous page, the type of YugiOh Card can be seen on the card itself in the caption box (as seen in the previous page). Although it would be simple to implement a text detector to read the text itself, it may not always be efficient since there it is only 1 word that needs to be read. The position of said word could also change with more complex cards.
For this case, color detection was implemented in order to determine the type of card that exists. The general outline of a card with each of their types labelled can be seen on the left.

How does it work?

The Function Call
1

Before anything, A 'mask' is applied to the image to cover the picture in the middle

The image is then converted from RGB to HSV (Hue, Saturation, Value)

These can be seen below

2

We define color boundaries for the color we want to detect in HSV notation

These Boundaries are then used to create a mask

This mask is dilated to bring out the color.

This is done by performing convolution (with the current image) with a 5x5 matrix of 1s

The result is then bitwise ANDED with the HSV image to extract only the desired colors

3

Count the mask with the most number of white pixels

That color boundary used for that mask corresponds to the card type
