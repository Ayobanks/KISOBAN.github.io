Attribute Detection

About

As seen in the previous page, this detected the attribute which is the circle on the top right of the card. This allows us to classify what kind of card is presented to the camera. All the different Attributes can be seen on the left. (*Note: there is also a 'divine' attribute but I decided to leave that since there are very few of those types of cards.)

How does it work?

The Function Call
1

Each of the attributes (as seen above) must be gray scaled and binary thresholded

The same is also done with the input image after its been resized

These can be seen below

Threshold
2

The input thresholded image is now bitwise ANDED with each of thresholded attribute images

Each produces a separate image result with a different number of pixels

These can be seen below

Bitwise AND
3

Count the number of White Pixels

The image with the most white pixels must match that of the original image

Therefore the number of white pixels is counted for each result

The image with the most white pixels determines which attribute that image belongs to

Count
Some Problems that occurred
Thresholding Problems

Some images were already of a light color so when thresholded would return an image that was white in areas other than the symbol so it was necessary to darken some images of the attributes so that when it is thresholded, only the symbol and letters would remain white
